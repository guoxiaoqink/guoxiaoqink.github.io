git代码部分不提交

发表于 2018-03-22 | 分类于 工具

Git Stash

git stash 可用来暂存当前正在进行的工作，比如：当前正在进行三个项目a、b、c. bc开发完了，a开发了一半。当你的老大让你提交代码的时候。你只想提交已完成的b和c。不想提交a。这时候你会怎么做呢？

你现在在dev分支，提交了bc之后，需要切换到正式的分支把bc也提交。这时候，如果不把a commit的话，你是不能切换分支的。这时候，就可以先在dev分支上执行 git stash 。然后切换到正式分支提交bc。然后回到dev git stash pop继续开发。

进阶：

当你多次使用’git stash’命令后，你的栈里将充满了未提交的代码，这时候你会对将哪个版本应用回来有些困惑，

git stash list命令可以将当前的Git栈信息打印出来，你只需要将找到对应的版本号，例如使用’git stash apply stash@{1}’就可以将你指定版本号为stash@{1}的工作取出来，当你将所有的栈都应用回来的时候，可以使用’git stash clear’来将栈清空。

用github配置自动化测试部署项目

发表于 2018-03-22 | 分类于 CI

下边是一个工作流程实例

如果react App能够自动的测试和部署，这是不是挺令人惊讶的。通过github和Travis CI （CI是持续集成的意思）能实现这个目标

如果你符合下边的条件，那么本教程适合你:

  • 知道 如何构建一个javascript应用。本例子适用于react应用。所有遵循典型的测试和开发的应用，都可以在此步骤的基础上进行调整
  • 知道 什么是单元测试.
  • 相当的熟悉 git和github
  • 熟悉 运用命令行
  • 不知道 不知道如何设置自动化测试和开发
  • 需要 需要免费的主机去展示你的javascript项目

本向导将告诉你如何创建一个简单的react的应用，将它推到GitHub，链接到Travis CI 再到 GitHub pages。本教程不关注什么是react

步骤1、创建一个JavaScript应用程序，添加测试。

为了达到这篇教程的目的。我将要使用create-react-app这个框架。这是一个快速创建react项目的一个工具。使用的这个工具创建的项目，里边已经包含了一个简单的单元测试用例

你已经安装了这个 NPM 和 node 运行这段命令 node -v 和 NPM -V检查是否安装成功。

运行安装程序npm install -g create-react-app安装create-react-app 这个在电脑上

  • 创建一个新的应用程序“create-react-app test-app”。这个命令需要一些时间。一旦完成，您的应用程序将位于test-app目录中。

  • 确保应用程序正常运行。完成后，进入到主项目的目录下，运行npm run start 或者 npm start`启动开发服务器。工作完成后，按Ctrl + C取消任务。

  • 确保npm run test成功运行。它应该有一个单一的测试套件。
    -确保npm run build运行成功。它应该在主目录的/build文件下创建了一系列的东西

步骤2、创建一个GitHub库，将你的项目推上去。

在GitHub上创建一个没有任何文件的新的库空库。确保为您的机器添加了ssh密钥，然后将项目文件全部放到这个库中：

刷新你的github账户，确保本地文件全部推送的github

步骤3、生成一个GitHub的token。

On GitHub, 去你的个人账户的设置界面 -> Personal Access Tokens.

生成一个新的token。token是访问github库的一种钥匙。将token保存起来，既然是访问修改你github上库的一种钥匙，好好保存撒。token将在下一节中使用。

  • 备注 ：token不应该与任何人共享，它本质上提供了一个入口密钥访问来编辑您的库。

步骤 4、 在你的项目中使用Travis CI,使用的方式就是在项目的根目录下创建 .travis.yml 文件.

点击这个链接create a Travis CI profile ，进入Travis官网，用你的GitHub账户登录，登录后，选择用Travis 的GitHub库，将该库选中。选中的库由灰变为绿色。

在创建这个文件之前,在Travis中，打开选中的这个库，打开这个库的右上角的设置. 去这个项目的设置界面 。 添加一个新的环境变量命名为 github_token, 将这个变量的值设置为上边的token. 这样Travis将会用这个token访问你的库

在项目的根目录下，创建名字为 .travis.yml 的文件

每一句代表的意思是:

告诉 Travis 使用 NodeJS.

告诉 Travis 使用的NodeJS 的版本. 这里可以改变使用node的版本。比如 - "7" or - "6" .

告诉 Travis ：在每一次提交新的代码，项目重新build的、项目重新编译的时候。 缓存 node_modules 文件，而不是重新编译整个项目。

重要的一点: 告诉 Travis运行的命令行. , 如果该命令运行失败 (除了0以外的任何东西), 构建失败. 失败后不会部署.

部署结构. 有关其他部署平台的详细文档可以在这里找到 here.

provider: pages 指定我们使用GitHub Pages

skip_cleanup: true 告诉Travis不要删除生成的文件

github_token: $github_token 使用GitHub token。使用名字叫github_token的token。在上边我们在Travis的设置中配置过的。

local_dir: build 告诉 Travis 使用build下的文件进行部署

on: branch: master 告诉 Travis ：我们仅仅在改变master主分支代码的时候的时候，触发部署

步骤 5. 在package.json设置主页地址

https://username.github.io/project_name/

按照上面的语法，编辑您的文件以添加项目的URL的一个新条目：

“homepage”：“https://username.github.io/project_name/”

步骤6、commit push 代码。

添加新文件，travis.yml 到Git，将它推到GitHub库。travis将自动开始构建、测试和部署它。

Travis将排队您的项目，并尽快建成它。 home page。一旦测试和构建  项目完成。在GitHub库上就新创建了一个名为“gh-pages”的新分支。
